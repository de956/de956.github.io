Рекомендуется выбирать популярные дистрибютивы GNU/Linux для первого опыта использования!

Такие как Ubuntu (Lubuntu/Kubuntu/Xubuntu), Linux Mint, Fedora (Debian и Arch тоже популярны, но более сложны в настройке/установке).
Эти дистрибютивы с доступной документацией и большим сообществом пользователей. Поэтому, в случае каких-либо вопросов, скорее всего, ответ будет на первой поисковой странице, либо всегда будет возможность спросить у многочисленных пользователей.
Рекомендуется устанавливать программы из репозиториев дистрибютива или сайта/репозитория конкретного производителя программы.
Рекомендуется использовать менеджеры версий Ruby, такие как rvm/rbenv/chruby/asdf и не рекомендуется использовать "системный" Ruby дистрибютива.
Не использовать sudo/root для установки gem'ов, версий Ruby.
Не использовать директорию /tmp - это каталог временных файлов. После перезагрузки файлы будут удалены!
С осторожностью использовать команду удаления rm -rf.
Заглядывать и читать в директории /var/log/* - там системные логи.
Для справочной информации обычно достаточно добавить -h/-help/--help. Например: user@rubyschool:~$ ruby -h.

Можно также воспользоваться утилитой man, например: user@rubyschool:~$ man ruby.
Для поиска программ (например консольного файлового менеджера midnight-commander) из репозиториев дистрибютива можно воспользоваться командой:

для debian-based дистрибютива: user@rubyschool:~$ apt search mc

для arch-based дистрибютива: user@rubyschool:~$ pacman -S mc / user@rubyschool:~$ pacman -Ss mc.
Узнать место/каталог установленной программы: user@rubyschool:~$ which ruby.
Базовый список других команд: https://wiki.debian.org/ru/ShellCommands .
Не рекомендуется под root'ом работать и ходить на сервер по ssh.
Ни стоит (!) менять локаль/язык на сервере - оставить по умолчанию (т. е. EN/US, на латинице).
